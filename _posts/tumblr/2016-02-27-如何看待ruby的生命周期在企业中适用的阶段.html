Aaron: 目前Ruby 没有特别大的团队在推，团队招人也比较困难。如何看待Ruby在企业发展过程中适合的阶段和发挥的作用，以及Ruby语言的生命周期？

Vincent: ruby依然是小众的主流语言，适合初创阶段的公司使用，如果有稳定的技术大牛和团队撑场，可以一直使用。

说 ruby 小众是因为用的人实在太少，说 ruby 主流是它还生命力很强的语言，有它特别擅长的领域，有一批忠实拥趸，开发语言有它一席之地。
ruby 特别适合初创阶段使用，是由于它高开发效率，成熟领先的 rails 开发框架决定，但是想要长久使用，需要有能驾驭它的人，所以要有技术牛人和团队，能不能招到人留着人，就是最大的问题了。

ruby 性能问题的确让人不爽，我觉得应该这样看，大部分情景下语言性能本身的影响很小，如果关键情景有影响，完全可以用其它语言替代，象我们并不排斥 java c 和 golang. 而且目前 ruby 和 rails 进入相对成熟阶段，生态系统是很完整的。当系统达到一定规模之后，ruby 和 rails 带来开发效率上的提升已经没那么明显了，因为这个时候更多是架构层面的问题，无论什么语言挑战是一样的。所以后期还用不用其实是取决于团队的喜好。

ruby 属于那种上手十分容易，但是精通十分困难的语言，另外因为十分灵活强大，对人要求很高。所谓能力越大，责任也越大。所以团队必须有牛人领衔，不然很容易走歪。团队更适合小规模作战，以前超过5人的 ruby 团队就很大了。通过微服务化，划分清晰的系统边界，可以由多个小团队组成大规模团队，这方面我也在探索。

所以 ruby 小众但依然主流，适合能够驾驭它的团队。
